肌診断アプリ（SkinAI）

個人開発している美容 × AI アプリです。  
React Native（Expo）と Firebase を使用し、  
肌の写真からAI風の肌タイプ分析、スコア診断、履歴保存ができる仕組みを実装しています。

---

本アプリの動作デモでは、プライバシー保護のため 実際の顔写真ではなく、似顔絵の画像 を使用しています。
アプリの画面遷移および Firebase へのデータ保存の流れが確認できます。
動画内の画面遷移：
１．Firebase に診断データが存在しない状態
２.アプリのホーム画面
３．写真選択（似顔絵を使用）
４．肌に関する質問回答
５．診断データを保存
６．診断履歴画面の表示
７．診断詳細画面
８．irebase にデータが保存された状態の確認

目的
「美容で人に希望を与えたい」という思いから開発したアプリ。  
肌の悩みを可視化し、改善のきっかけをつくることを目指しています。
---

　使用技術

- React Native (Expo)
- TypeScript
- Zustand（状態管理）
- Firebase Authentication
- Firebase Firestore
- Expo Image Picker
- Expo Router

---

使用技術と選定理由

 React Native (Expo)
- クロスプラットフォームで iOS / Android を一つのコードで開発可能
- Expo により環境構築が簡単、個人開発でもスピーディに実装できる
- カメラ、画像処理など美容アプリに必要な機能が豊富

➡ 短期間でMVPを作るのに最適なため採用

---

 TypeScript
- 型安全によりバグを減らし、ロジック（肌診断の計算など）が安定する
- 将来の機能追加（AI導入、API拡張など）にも強い

➡ 拡張性と保守性を高めるため採用

---

 Zustand（状態管理）
- Redux よりシンプルで軽量。React Hook ベースで扱いやすい
- グローバル状態（ユーザー情報・診断データ）を直感的に管理できる
- パフォーマンスが高く、コードの可読性も良い

➡ 小規模～中規模アプリに最適な状態管理のため採用

---

 Firebase Authentication
- メール・パスワードや匿名ログインが簡単に実装できる
- サーバー不要で安全なユーザー管理が可能
- Expo と相性が良く、習得コストが低い

➡ ユーザー管理を最速で構築するため採用

---

 Firebase Firestore
- NoSQL でユーザーごとに違うデータ構造を柔軟に保存できる
- 診断履歴の保存・取得に最適
- リアルタイム更新が可能で、UX を高められる

➡ 診断データの保存に最も適したデータベースのため採用

---

 Expo Image Picker
- 写真撮影・写真選択を簡単に実装できる
- Expo SDK として提供されているため依存が少なく安全
- 肌診断の入力として「写真アップロード」が必須のため最適

➡ 美容診断アプリに必須の画像入力機能のため採用

---

 Expo Router
- ページ遷移がファイルベースで直感的
- Web のように画面構造が管理でき、コードがきれいになる
- 認証後の画面遷移（ログイン → ホームなど）を実装しやすい

➡ 画面遷移の管理を楽にして開発効率を上げるため採用

　主な機能
